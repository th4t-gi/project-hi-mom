### What kinds of tests are they?

These tests are testing both the front end API functionality, that we are parsing the REST JSON correctly, and also that all steps that need to be completed on the backend when an api endpoint is touched are executing correctly as well. When `npm test` is ran, we spin up the backend with a `E2E_TESTING` environment variable flag which creates a temperary test SQLite database to run our tests.

### For code that is not covered by tests, why did you not cover them?

The only part of the API pipeline that isn't covered with these test is simulating the "Sign up" button press that calls `createUser` for example. This was not covered mainly because to test this would require setting up some sort of automated web interaction pipeline with Playright which was too labor intensive for the marginal coverage it would've given us.
