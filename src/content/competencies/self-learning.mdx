### Describe the technology that the tutorial addresses and how it fits into your project.

The tutorial that I gave was on the ReactJS front end framework. In almost any modern large scale web application, the engine behind the user interface is React. This is because of React's Virtual DOM and component architecture allows user interfaces to be built with a focus on state changes as opposed to events and abstracted away the smaller elements in a user interface to better align with the mental model that developers use when building applications. Because of all of these benefits, we used ReactJS with Typescript for the front end of our tech stack.


### Evaluate how useful the tool was in your work. What were its strengths and weaknesses, especially in comparison with other tools you have used?

React is indispensable for our project. Allowing for dynamic state changes and providing a large ecosystem of UI Frameworks (i.e. Card, Button, and Dialog components) simplified both our engineering and design processes. Looking back, our front end is not super heavy or in need of real-time updates, so we probably could've gotten away with a static front-end like NextJS, but a strength of React is that it is highly scalable, so if we want to add some crazy livestreaming feature or something, we have the foundation to do so.

### Identify one particular sticking point to using this technology that you would want your team to know about when adopting the tool and how you resolved it.

I think one thing that people don't realize when they are new to React is that the components refresh on *every* event that happens, which is why it is very important to use hooks like `useEffect` and `useMemo` to ensure you aren't calling an API every time the mouse moves.
